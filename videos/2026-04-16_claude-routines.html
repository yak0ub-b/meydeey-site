Les Routines Claude sont des configurations sauvegardées (fiche de poste + clés d'accès + déclencheur) qui tournent automatiquement — localement via Claude Desktop ou dans le cloud via Claude Code. Conçues pour les entrepreneurs et professionnels ayant des tâches répétitives : veille, reporting, triage de CV, mise à jour de repos.

Synthèse profonde

L'image mentale proposée est le "stagiaire qui vit dans le cloud" — très compétent, invisible, disponible 24/7, qu'on ne verra jamais. On lui donne trois choses : une fiche de poste (ce qu'il doit faire), des clés d'accès (Gmail, Notion, Drive, Slack…), et un déclencheur (quand se mettre au boulot). Il se met au travail même quand l'ordi est éteint.

Trois types de déclencheurs : (1) réveil matin = schedule cron (tous les jours à 8h, chaque lundi…), (2) sonnette = API/webhook (un formulaire rempli déclenche le run), (3) détecteur de mouvements = GitHub (une PR ouverte, un push…). Pour les débutants, seul le schedule est accessible sans expertise technique.

Deux modes d'exécution. Local (Claude Desktop) : accès aux fichiers locaux, données sensibles non transmises au cloud, parfait pour contrats/factures/pièces client. Distant (Claude Code + GitHub) : repo comme source de vérité, exécution dans le cloud d'Anthropic, parfait pour trier des CV envoyés par email, mettre à jour des repos, faire de la veille.

Piège pricing : le plan Team standard à 25$/siège ne donne pas accès aux Routines — il faut le plan Team Premium à 100$/siège. 100$/mois = 14× moins cher qu'un SMIC. Les données de la routine tournent dans le cloud d'Anthropic — contrats confidentiels, bulletins de paie, liasses fiscales doivent rester en local (Claude Desktop local uniquement).

Méthode — Créer une routine locale (avocat)

  1. Ouvrir Claude Desktop → icône Code → section Routines → Nouveau
  2. Choisir "Local" (fichiers sur ta machine)
  3. Renseigner : nom, description, prompt détaillé (sources, domaines, périmètre, format de sortie)
  4. Sélectionner le dossier client auquel donner accès
  5. Choisir la fréquence (quotidien à 9h)
  6. Activer "accepter modifications automatiquement" si le périmètre est bien défini
  7. Créer → tester avec "Exécuter maintenant" → vérifier l'historique

Concepts clés

Claude Routines
Configurations Claude sauvegardées = fiche de poste (prompt) + clés d'accès (outils connectés) + déclencheur (schedule/API/GitHub). Tournent automatiquement sans intervention humaine. Disponibles dans Claude Desktop (local) ou Claude Code (cloud, nécessite GitHub). Accessibles uniquement sur le plan Team Premium (100$/siège/mois).
Pourquoi c'est important : Première fonctionnalité officielle d'Anthropic pour l'automatisation récurrente sans code — accessible aux non-développeurs via Claude Desktop. Équivalent d'un agent 24/7 mais piloté par des configurations sauvegardées, pas du code.
Local vs Distant — deux modes d'exécution
Local (Claude Desktop) : les fichiers restent sur ta machine, Claude y accède pendant l'exécution — idéal pour données sensibles (contrats, bilans). Distant (Claude Code + GitHub) : le repo est la source de vérité, Claude tourne dans le cloud d'Anthropic, laptop peut être éteint — idéal pour CV par email, veille, monitoring de repos.
Pourquoi c'est important : La distinction local/distant détermine ce qui peut être automatisé légalement et sans risque de fuite. Confondre les deux = risque RGPD réel sur des données sensibles.
GitHub comme source de vérité pour les routines
Pour les routines distantes, le repo GitHub contient : le contexte de l'entreprise, les fiches de poste, les données à traiter (CV, rapports), et les résultats produits par la routine. Claude lit, traite et re-push dans le repo — entièrement sans intervention humaine. Le repo est auditable, versionné, récupérable si le PC crame.
Pourquoi c'est important : Transforme GitHub en "base mémorielle d'entreprise" automatiquement maintenue — applicable même sans infrastructure cloud complexe.

3 cas métier concrets

Avocat — veille juridique quotidienne
Tous les matins à 8h : Claude scanne Legifrance, Dalloz, Village Justice → filtre les domaines du dossier client → envoie les 3 décisions clés par email. Gain : 45 min/jour. Le junior se concentre sur la plaidoirie, pas la veille.
Expert-comptable — reporting client automatique
Le 5 de chaque mois : Claude ouvre l'export comptable → calcule les écarts avec Python déterministe (pas d'IA pour les maths) → détecte les anomalies → prépare un PDF de 2 pages par client. Gain : 10h/mois sur 30 clients (20 min × 30).
DRH — préqualification CV
Les CV arrivent par email → automatisation → repo GitHub dossier "CV à traiter" → routine tourne la nuit → score sur 100, classement, rapport. Le lendemain matin : tout est dans "CV traités". La DRH contrôle, valide, envoie. Zéro lecture manuelle de CV hors-sujet.

Insights non-évidents

  • Le plan Team standard à 25$/siège ne donne pas accès aux Routines — piège courant. Il faut le Team Premium à 100$/siège. Vérifier avant d'investir du temps de setup.
  • Pour le reporting comptable, Claude utilise du code Python déterministe pour les calculs — pas de l'IA probabiliste. La routine garde son rôle de préparateur, le comptable garde son rôle de contrôleur.
  • L'automatisation email → GitHub → routine → résultats → rapport est possible sans coder : une automatisation n8n ou Zapier peut déposer les CV dans le repo GitHub avant que la routine s'exécute.
  • Claude est le meilleur sur les benchmarks de search/recherche web (au-dessus de Gemini, GPT-5.2, Grok) — c'est ce qui rend la veille juridique particulièrement fiable avec les Routines.
